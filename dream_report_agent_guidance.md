# DREAM DOWNLOAD REPORT GUIDANCE 

## Core Principle

The Executive Summary and Solution Summary must directly answer the
original workshop objective.

If these sections do not clearly address the workshop ask, the report
has failed.

All other sections in the report exist only to provide evidence
supporting that answer.

------------------------------------------------------------------------

## Primary Rule

The DREAM report must always answer the question the organisation asked
when commissioning the workshop.

The Executive Summary must provide the answer.

The Solution Summary must provide the recommended way forward.

All other sections provide supporting evidence.

------------------------------------------------------------------------

## Agent Reasoning Sequence

The agent must reason in the following order:

1.  Identify the Workshop Ask
2.  Determine the Key Insight that answers that ask
3.  Write the Executive Summary explaining the answer
4.  Provide Evidence from workshop signals
5.  Identify the underlying causes
6.  Recommend the Way Forward in the Solution Summary

------------------------------------------------------------------------

## Structural Logic

Workshop Ask\
↓\
Executive Answer\
↓\
Supporting Evidence\
↓\
Root Causes\
↓\
Recommended Way Forward

------------------------------------------------------------------------

## Executive Summary Guidance

The Executive Summary must:

-   Restate the workshop objective
-   Present the key insight revealed during the workshop
-   Explain why the insight matters to leadership
-   Describe the opportunity or risk revealed

The Executive Summary must clearly answer the workshop ask.

------------------------------------------------------------------------

## Evidence Guidance

Evidence must come only from workshop signals such as:

-   Discovery insights
-   Domain scores
-   Participant signals
-   Identified constraints
-   Perception vs operational gaps
-   Organisational friction indicators

Evidence must support the Executive Summary conclusion.

If evidence does not support the workshop objective or executive
insight, it should not be included.

------------------------------------------------------------------------

## Root Cause Analysis

The agent must analyse why the issues revealed in the workshop exist.

Possible categories include:

-   Technology fragmentation
-   Organisational misalignment
-   Capability gaps
-   Governance limitations
-   Operational complexity
-   Data limitations

Each root cause must be supported by workshop evidence.

------------------------------------------------------------------------

## Solution Summary Guidance

The Solution Summary must answer:

Given the workshop objective and the insights revealed, what is the
recommended direction forward?

The recommendation should:

-   Address the root causes identified
-   Align with organisational readiness
-   Reflect the insights surfaced in the workshop
-   Provide a logical path forward

The solution summary should describe the transformation direction rather
than a detailed implementation plan.

------------------------------------------------------------------------

## Evidence Principle

All sections other than the Executive Summary and Solution Summary exist
to support the central insight.

Evidence should:

-   Explain why the conclusion was reached
-   Demonstrate the patterns observed in the workshop
-   Reinforce the recommended direction forward

------------------------------------------------------------------------

## Validation Test

Before completing the report, the agent must test the following:

If an executive reads only the Executive Summary and Solution Summary,
will they clearly understand:

-   The answer to the workshop objective
-   The opportunity or risk revealed
-   The recommended direction forward

If the answer is no, the report must be regenerated.
