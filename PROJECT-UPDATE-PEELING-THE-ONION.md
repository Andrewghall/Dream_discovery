## What This Is

A fundamental redesign of the Cognitive Guidance live session page. Replaces the old "cycle through prep questions one at a time" model with a two-level hierarchy:

1. **Main Questions** — ~5 big questions per phase (REIMAGINE / CONSTRAINTS / DEFINE_APPROACH), displayed one at a time in a warm amber card. Facilitator navigates sequentially with Previous / Next buttons.
2. **Sub-Post-Its** — Dynamic lens-coloured sticky pads beneath the main question that explore specific angles within its scope.

**The concept:** Each main question gets deeply explored through dynamic sub-questions before moving on. Agents must stay goal-oriented — every sub-question must serve the current main question. Think of it as following breadcrumbs — the conversation may wander briefly, but the agents pull it back towards the destination.

### Sub-Post-It Sources (in priority order)
1. **Prep-generated starters** — 2-3 sub-questions per main question, generated by the Question Set Agent during prep, grounded in research/Discovery data. Ready to trigger dialogue immediately when a main question activates.
2. **Agent-generated** — The Facilitation Agent creates new sub-post-its during the live session, scoped to the current main question, following the team's conversation breadcrumbs.
3. **Signal-generated** — The existing deterministic pipeline continues producing signal-based pads.

### Layout Structure
```
┌─────────────────────────────────────────────────────────────┐
│            Main Question Card (full width, amber)            │
│  REIMAGINE                                    Question 2 / 5 │
│  "If we could create the perfect customer experience..."     │
│  Lenses: ● People  ● Organisation  ● Customer               │
│         [← Previous]                    [Next Question →]     │
└─────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────┬──────────────────┐
│                                          │  Covered (4)     │
│  Active Sub-Post-Its                     │                  │
│  ┌────────┐ ┌────────┐ ┌────────┐       │  ▶ Q1 (6 subs)  │
│  │ People │ │ Org    │ │ Custom │       │  ▼ Q2 (4 subs)  │
│  │  62%   │ │  35%   │ │  10%   │       │    ✓ People...  │
│  └────────┘ └────────┘ └────────┘       │    ✓ Org...     │
│                                          │    ✓ Customer.. │
└──────────────────────────────────────────┴──────────────────┘
```

### Key Rules
- Coverage threshold: **70%** for auto-marking sub-pads as covered
- `currentMainQuestion` synced to server so agents can read the current goal
- Agents MUST stay within scope of the current main question
- Collapsible history: completed sub-pads grouped by main question in accordion sections
